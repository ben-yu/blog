SpaceBees

A HTML5 Multiplayer Shooter

A Browser-based multiplayer game using Three.js, Socket.io, and Node.

Entirely built using Coffeescript for that much needed syntatic sugar and readability. Using Backbone for the frontend framework simplified syncing with the backend, and a bunch of other things like event binding etc... I could also share the same models and collections with the Node backend.

Brunch provided an amazing build pipeline, automatically compiling and optimizing the entire application for deployment. Socket.io provided realtime communication with the backend via HTML5 Websockets.

It is still a work in progress...

Forward

A thin e-mail client for small business

E-mail is the core of business communication, yet it still remains a passive form of communication. It is still paper mail, just in electronic form. Forward is a reimagining of e-mail, brining a suite of communication tools to improve and enhance e-mail as a core business communication tool.

We developed an initial prototype using Ember.js with Node.js backend to handle e-mail and metadata.

Sones

An iOS play-based audiometric hearing diagnostic app

Sones is a game app that allows the user to play a collection of games with acoustic ques which first diagnoses the hearing ability of the user and potentially enables the user to overcome their hearing issues by better adapting to the game. This project is part of ongoing research at APL The sound-based games that measures the player’s hearing ability in the form of an pure-tone audiogram. A diagnosis is arrived by compared player performance to common audiogram shapes using mean square error. Player data sent remotely to a web server allowing for remote diagnostics

Facial Recognition

Machine Learning Final Project

TTC Signage Proposal

Updating TTC signage to improve usability for visual/hearing impaired

  • Identified signage problems in the TTC, and proposed improvements such as: strategic sign placement, improved color contrast, tactile feedback system on handrails and floor tiles, and consistent font selection
  • Produced scale mock-up of St. George Station with proposed signage changes
  • Created surveying program to evaluate our new system compared to current signage system

Bridge Design

Pedestrian bridge and scaled mock-up cardboard bridge

  • Collaborated with 3 team members to design and build a model railway bridge that would safely support the passage of a 400N train, while restricted to using one sheet of mat-board 32" x 40" x 0.05" and contact cement
  • Final bridge design withstood approximately 500N of force
  • Design accounted for axial and flexural stresses (compressive and tensile), shear stress, beam deflection, and plate buckling
  • Wrote 12 page engineering design report explaining our design process, justifying our design decisions and provided grounds in the form of calculations and drawings
